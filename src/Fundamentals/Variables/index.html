Variables:
- Can hold a string value at one moment then int in another
Why ? JS is dynamically typed

Number:
Number can represent an int or float point number

BigInt:
the "n" at the end means it's a BigInt
Ex: const bigInt = 1234567890123456789012345678901234567890n;

String
3 Types of Quotes
Double quotes: "Hello".
Single quotes: 'Hello'.
Backticks: `Hello`.
Note: Backticks are extended functionality quotes. Allow us to embed variables and expressions
into String by wrapping them in ${}
There is NO char type in Javascript

Boolean
Either true or false

Null value
Special value which represents Nothing
Ex: let age = null

Undefined value
Means "Value is not Assigned"
Ex: let age .... alert(age)
Undefined is reserved as default initial value for unassigned things

Objects and Symbols
Symbols is used to create unique identifiers for objects

Tip:
Use the typeof(x) operator to find out what type a variable is
